hexo配合githubpages搭建免费博客

总思路

  • 使用gitpages当免费的网页空间和使用域名
  • 本地搭建hexo网站，并使用hexo的自动工具发布到git
  • 另外建个git仓库存放笔记，配置obsidian自动拉取笔记和上传笔记
  • 笔记积累多了需要发布，从git上拉取下来，覆盖到hexo的/source/_post文件夹下覆盖
  • 执行hexo clean命令清理缓存，执行hexo g 命令产生静态HTML，执行hexo d发布到git
    也可以把obsidian、hexo这些集中到一个git仓库里，需要建俩分支，但是我经过思考还是分开比较方便，因为博客仓库不用反复提交代码了，还不用另外创建分支了，还能做一些权限控制

总过程

  • 注册git账号，git地址：https://www.github.com (注册时需要一个邮箱，邮箱要可以接受验证码)
  • 在git新建仓库，以git用户名为前缀，比如username.github.io，仓库是public权限，不是private权限
  • 在本地安装git，配置需要当笔记仓库的git，建议配置ssh上传文件
  • 本地安装nodejs和hexo
  • 在本地新建hexo站点，并安装设置主题
  • 本地站点发布到git的main分支
  • 使用obsidian打开本地博客，进行页面编辑

涉及的hexo插件

  • hexo-auto-category：自动依据文件夹名称给文章添加分类标签，节省很大工作量
  • hexo-deployer-git：使用hexo命令自动发布到git

涉及的obsidian插件

  • obsidian git
    （本插件需要你在本地电脑文件夹里配置好git的用户名、邮箱、远端地址、密钥等才能正常使用）

重要配置

安装nodejs和hexo

  • 下载nodejs 地址是：https://nodejs.org/zh-cn/download，然后安装
  • 打开windows命令行，进入pbsite文件夹
  • 输入命令npm install -g hexo-cli
  • 然后输入 hexo init pbsite(pbsite就是你的博客的总文件夹，后面会使用git命令把它改成一个git可以控制的仓库，不会影响博客代码和文件)
  • 然后输入cd pbsite就进入这个文件夹了，在这个文件夹里输入npm install 就开始安装博客代码了，结束以后这个文件夹里就是博客的各种文件夹和代码
  • 然后安装fliud主题，在pbsite文件夹里执行npm install–save hexo-theme-fluid，这样就在pbsite/node_modules文件夹下能找到这个主题了
  • 然后安装自动分类插件，在pbsite文件夹里执行npm install–save hexo-auto-category，这样就在\pbsite\source_posts文件夹下你建的文件夹就能自动生成博客的分类，笔记和分类就是在这个文件夹里面写。
  • 然后安装hexo的git插件，这样hexo就可以把代码自动部署到github远程的仓库，命令是npm install–save hexo-deployer-git，也是安装到pbsite/node_modules文件夹，通过这个插件，使用hexo d命令就能把博客上传到远程git仓库，要想上传还需要告诉hexo上传到哪，需要在pbsite文件夹下的_config.yml文件里配置，就在文件的最下面4行，改成如下：

git仓库的搭建

  • 准备一个能收验证码的邮箱注册git账号，git地址是https://github.com/ ，注册好以后新建仓库，仓库名是git账号的用户名+github.io，比如username.github.io，注意是public权限，我记得private权限也可以，不过我没试。
  • 然后本地安装git，下载地址是https://git-scm.com/downloads ，mac应该自带了不用安装，或者需要安装的话搜索一下安装就行，win11的话直接一条命令winget install git也可以安装好，或者安装git的桌面端也可以
  • 电脑上安装好git以后，打开win的终端
  • 通过cd命令进入到hexo新建的pbsite文件夹，执行git init命令，这样这个文件夹就初始化成本地的git仓库了
  • 然后在本地配置git的用户信息，git config –global user.name ‘你注册的git的用户名’，git config –global user.email ‘你注册git的email’
  • 然后使用ssh命令生成ssh密钥，命令是ssh-keygen -t rsa -C ‘你注册git的邮箱’，然后出现询问的地方全部回车，结束以后会在c盘-用户-administrator下出现一个.ssh文件夹，里面有id_rsa和id_rsa.pub两个文件，id_rsa.pub用文本编辑器打开复制里面的内容，然后打开github登陆，点击右上角的头像，选择settings，再点击左侧栏里的’SSH and GPG keys’选项，把刚才复制的id_rsa.pub里的内容粘贴进ssh keys那个文本框里，然后再title文本框里输入一个名字点保存。参考：http://www.wenwl.site/pages/f0c240/#%E9%85%8D%E7%BD%AEssh
  • 回到电脑终端，输入ssh -T git@github.com，能连上git会提示你输入yes，输入以后就配置好ssh了

下载和使用obsidian

  • obsidian可以在所有平台系统使用，比如win，linux，安卓，苹果，而且obsidian可以使用github仓库当云存储
  • obsidian的下载地址：https://github.com/open-source-labs/obsidian/releases/下载安装以后
  • 把\pbsite\source_posts文件夹里的内容全复制到一个文件夹里，比如叫notes,这个notes文件夹建哪个磁盘都行，给obsidian当笔记数据库用的
  • 然后打开obsidian，在界面里会让你选择是打开本地的，还是同步远程的还是新建数据库，选择打开本地文件夹，选择上面建的notes文件夹打开，这样notes文件夹就是一个obsidian的笔记库了
  • 然后再在github.com上再用自己的账号新建一个仓库，设置为private权限，仓库名字随便起吧，这个仓库作为obsidian日常同步笔记使用的
  • 然后在电脑上进入notes文件夹，执行’git remote add url 你上面新建的仓库地址’，如果命令不成功，就在notes文件夹下分别执行git init和git add -all和git commit -m ‘注释’几个命令，这样notes文件夹就和远程的仓库连接了，obsidian git插件就可以自动往远程同步文件了。
  • 然后给obdisian软件安装‘obsidian git’插件，它还需要安装一个插件具体安装和设置教程百度吧
